JEB confirms that WebFragment.addMiFiJsInterfaces() injects MiFiJsBridge, and MiFiWebViewClient.interceptRequest() handles localresource:// by taking Uri.getPath() and passing it to FileUtils.readFileAsStream(path). It returns new WebResourceResponse("", "", stream) without explicit CORS headers. So if this PoC is loaded over http(s), JS often cannot read the body directly. To dump bytes into the DOM reliably, relaunch this same PoC via localresource://x/... so the page itself becomes same-origin with the target resource.
